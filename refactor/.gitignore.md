**Date:** 2026-03-20
**Refactored by:** Claude Sonnet 4.6

## File Overview

| Property | Value |
|---|---|
| **Path** | `.gitignore` |
| **Purpose** | Prevents secrets, generated output, and tooling artifacts from being committed |
| **Role** | Project-wide gitignore — applies to all contributors and CI |

### Section inventory (before refactor)

| Section | Patterns | Coverage |
|---|---|---|
| Secrets | `.env`, `.env.*`, `!.env.example`, `*.pem`, `*.key`, `certbot/` | API keys, TLS certs |
| Python | `__pycache__/`, `*.pyc`, `*.pyo`, `.venv/`, `venv/` | Bytecode, virtual envs |
| Claude / tool data | `.claude/`, `.superpowers/`, `.playwright-mcp/` | Local AI tool state |
| Generated output | `output/`, `backup/` | Pipeline outputs, DB dumps |
| Nested repos | `deerflow/` | Archived subproject |
| Jupyter | `.ipynb_checkpoints/`, `notebooks/.ipynb_checkpoints/` | Notebook checkpoints |

## Gaps Found

| # | Gap | Severity | Description |
|---|---|---|---|
| 1 | `logs/` not ignored | **Medium** | `logs/nginx-private/access.log` and `logs/nginx-private/error.log` were untracked. Log files can grow large, contain PII (IP addresses, request paths), and have no value in git history. |
| 2 | `bitwarden_import*.json` and `bitwarden_import*.csv` not ignored | **Medium** | A `bitwarden_import.json` was present at the repo root (untracked). Bitwarden import files can contain plaintext vault entries. Pattern added to prevent accidental commit. Note: `_archive/bitwarden_import.csv` was already committed before this pass and cannot be retroactively hidden by .gitignore alone. |
| 3 | `ssl/` not ignored | **Low** | On the VPS, `./ssl/` holds the webroot-issued TLS certificate tree (`fullchain.pem`, `privkey.pem`, etc.). While `*.pem` and `*.key` patterns catch the cert files themselves, the directory and any non-standard file extensions could slip through. Adding `ssl/` closes the directory-level gap. |
| 4 | `_archive/output_project/` untracked | **Info** | Pipeline-generated project code is present in `_archive/output_project/`. Not added to .gitignore — archiving pipeline outputs may be intentional. User should decide whether to track or ignore. |
| 5 | `opportunities/` tracked in git | **Info** | `opportunities/done/url-shortener.json` is committed. The `opportunities/` directory holds pipeline opportunity records mounted into n8n. Tracking is intentional — these are structured data files, not generated artifacts. No change. |

## Changes Made

| Change | Before | After | Reason |
|---|---|---|---|
| Added `logs/` | *(absent)* | `logs/` | Nginx and app logs — large, contain PII, no value in git |
| Added `ssl/` | *(absent)* | `ssl/` | VPS TLS certificate directory — closes directory-level gap alongside `*.pem`/`*.key` |
| Added `bitwarden_import*.json` | *(absent)* | `bitwarden_import*.json` | Vault import data — sensitive, should never be committed |
| Added `bitwarden_import*.csv` | *(absent)* | `bitwarden_import*.csv` | Same reason; covers CSV format |
