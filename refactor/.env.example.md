# Refactor: .env.example

**Date:** 2026-03-20
**Refactored by:** Claude Sonnet 4.6

---

## File Overview

| Property | Value |
|---|---|
| **Path** | `.env.example` |
| **Purpose** | Documents all env vars required by the stack; copy to `.env` to configure |
| **Role** | Committed reference — `.env` is gitignored, this file is not |
| **Loaded by** | Manual: `cp .env.example .env` then fill in values |

---

## Gaps Found

| # | Gap | Severity | Description |
|---|---|---|---|
| 1 | `PORTAL_OIDC_CLIENT_SECRET` missing | **High** | `oauth2-proxy-portal` in both compose files requires this secret. Without it, the portal SSO proxy fails to start. |
| 2 | `N8N_PUBLIC_BASE_URL` and `N8N_PRIVATE_HOSTNAME` missing | **Medium** | `docker-compose.prod.yml` uses these var names for n8n's WEBHOOK_URL, N8N_HOST, and N8N_EDITOR_BASE_URL. Only the homelab names (`N8N_WEBHOOK_URL`, `N8N_HOST`, etc.) were documented. Someone setting up the VPS stack would set the wrong vars. |
| 3 | `NOTIFY_SMS_EMAIL` missing | **Medium** | `docker-compose.prod.yml` n8n uses `NOTIFY_SMS_EMAIL`; the homelab uses `NOTIFY_SMS_TO`. Both are valid, different env vars for the same functional purpose. Only `NOTIFY_SMS_TO` was documented. |
| 4 | Twingate vars missing | **Medium** | `TWINGATE_NETWORK`, `TWINGATE_ACCESS_TOKEN`, `TWINGATE_REFRESH_TOKEN` are required by the `twingate` service in `docker-compose.prod.yml`. Not documented. |
| 5 | `CF_API_TOKEN` missing | **Medium** | Required by `certbot-dns` for Cloudflare DNS-01 wildcard cert renewal (both compose files). Already flagged in `docker-compose.yml` refactor; now added to .env.example. |
| 6 | `OLLAMA_CLOUD_API_BASE` and `OLLAMA_CLOUD_API_KEY` commented out | **Low** | These vars ARE referenced in compose files (with defaults: `http://ollama:11434` and `ollama`). Having them commented implied they were optional; they should be listed as active (empty = use default). |
| 7 | Duplicate `# ANTHROPIC_API_KEY=` comment | **Low** | Line 9 had a commented-out duplicate of line 10. Removed. |
| 8 | Misleading header comment | **Low** | Header said "NEVER commit this file to git" — referring to `.env`, not `.env.example`. Header rewritten to clarify that `.env.example` IS committed; `.env` is not. |
| 9 | Higgsfield entries not in any compose file | **Info** | `# HIGGSFIELD ID=` and `# HIGGSFIELD API KEY SECRET=` commented entries removed — not referenced anywhere in the stack. |
| 10 | VPS-specific vars (`SDLC_SCRIPTS_DIR`, `SDLC_OUTPUT_DIR`, `SA_LOGO_PATH`) missing | **Info** | These have defaults in `docker-compose.prod.yml` so not strictly required, but worth documenting. Added as commented examples in the VPS section. |
| 11 | `WATCHTOWER_NOTIFICATION_URL` undocumented | **Info** | Commented in compose, documented as commented in .env.example with format hint. |
