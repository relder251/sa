3 — Pipeline Phases 1–10

Full pipeline sequence from webhook trigger through planning, code generation, testing, quality gate, documentation, human approval gate, git push, deployment, and monitoring with Slack notification.

flowchart TD
  TRIGGER["POST /run-opportunity\n{name, prompt}"]

  P1["Phase 1: Plan\nLLM: hybrid/chat\nGenerates project_plan.md\nArchitecture, stack, file structure"]

  P2["Phase 2: Code Generation\nLLM: free/code or cloud/code fallback\nOutputs all files in FILE block format\nRequires: main app, tests/, requirements.txt, Dockerfile"]

  P3["Phase 3: Format Validation\nLLM: free/code\nValidates FILE block syntax\nAuto-fixes malformed output"]

  P4["Phase 4: Extract and Test Loop\nExtracts files to project_dir\nRuns pytest in isolated venv\nLLM fix loop up to MAX_ATTEMPTS\nPostprocesses requirements.txt"]

  P5["Phase 5: Quality Gate\nRuns ruff, bandit, mypy\nLLM security review if HIGH+ bandit issues\nCan BLOCK pipeline on critical vulnerabilities"]

  BLOCKED{"Phase 5\nBLOCKED?"}

  P6["Phase 6: Documentation\nLLM generates README.md\nCHANGELOG.md and API docs\nModel: cloud/fast"]

  P10GATE{"ENABLE_APPROVAL_GATE\n= true?"}

  P10["Phase 10: Human Approval Gate\nSends Slack notification with approve/reject URLs\nWaits for POST /approvals/run_id/signal\nTimeout: APPROVAL_TIMEOUT_HOURS default 24h"]

  APPROVED{"Approved?"}

  P7["Phase 7: Git Push\nInitializes git repo in project_dir\ngit add, commit, push to GIT_REMOTE_URL\nOptionally builds and pushes Docker image to DOCKER_REGISTRY"]

  P8["Phase 8: Deployment\nDEPLOY_TARGET options:\nskip default - no deploy\nlocal - docker run on host\nssh - scp and docker compose on remote host"]

  P9["Phase 9: Monitoring and Notify\nHealth checks against endpoint/health\nUp to 5 checks with 10s interval\nSlack webhook notification via SLACK_WEBHOOK_URL"]

  DONE["Pipeline complete\nrun_state.json updated\nstatus: done"]

4 — SSO Auth Flow

Browser request through nginx to oauth2-proxy using PKCE S256, Keycloak OIDC token issuance, cookie validation, and upstream proxy to the protected service. Applies to n8n, LiteLLM, JupyterLab, WebUI, Portal, and Vaultwarden.

sequenceDiagram
  participant Browser
  participant nginx as "nginx-private\n(*.private domain)"
  participant proxy as "oauth2-proxy\n(PKCE S256)"
  participant kc as "Keycloak\nrealm: agentic-sdlc"
  participant svc as "Upstream Service\n(n8n / LiteLLM / Jupyter\nWebUI / Portal / Vaultwarden)"

  Browser->>nginx: HTTPS request to protected URL
  nginx->>proxy: Proxy pass to oauth2-proxy port

  alt No valid session cookie
    proxy->>Browser: 302 Redirect to Keycloak auth endpoint
    Note over proxy,kc: code_challenge=S256, PKCE flow
    Browser->>kc: GET /realms/agentic-sdlc/protocol/openid-connect/auth?code_challenge=...
    kc->>Browser: Login page (username + password)
    Browser->>kc: POST credentials
    kc->>Browser: 302 Redirect with authorization code
    Browser->>proxy: GET /oauth2/callback?code=AUTH_CODE
    proxy->>kc: POST /protocol/openid-connect/token (PKCE code_verifier)
    kc->>proxy: access_token + id_token + refresh_token
    proxy->>Browser: Set-Cookie session (encrypted, secure=true)
  end

  alt Valid session cookie present
    proxy->>proxy: Validate session cookie
    proxy->>svc: Proxy request with X-Auth headers
    svc->>proxy: Response
    proxy->>Browser: Response
  end

  Note over proxy: OIDC_ENABLED=false gate bypasses auth for lead-review
  Note over proxy: n8n skips auth on /webhook/ and /webhook-test/ routes
  Note over proxy: Keycloak role check: agentic-sdlc:admin or agentic-sdlc:user
